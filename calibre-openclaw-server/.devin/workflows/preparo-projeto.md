# Project Setup Workflow

This workflow configures the initial project structure by creating essential
folders and the documentation needed for organization and development.

## Overview

The setup workflow creates:

- `scripts/` for Python scripts;
- `library/` for reference documents and books;
- `venv/` for the Python virtual environment;
- the main `README.md` with project information;
- library documentation when files exist;
- Markdown summaries for each PDF when PDFs exist.

## Usage

Type `/project-setup` to run the setup workflow.

## Workflow

1. Create essential folders.
2. Create or update the root `README.md`.
3. Document library files when they exist.
4. Create Markdown summaries for PDFs when they exist.

## README Requirements

The main README must include badges, an animated header, project objective,
folder structure, animated footer, final summary, and changelog.

## Important Notes

- `venv/` must be created but not activated automatically.
- The virtual environment must be activated manually by the user.
- PDF summaries must be created only when PDFs exist.
- If the library is empty, its README may be omitted.
- All Markdown files must follow the documentation rules.

## Next Steps

1. Activate the virtual environment.
2. Install required dependencies.
3. Start development using the PDCL methodology.
